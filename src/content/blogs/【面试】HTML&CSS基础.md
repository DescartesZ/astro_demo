# 一、HTML&CSS基础
前端页面渲染的根基，是所有可视化开发的前置知识，分为 2 个二级模块。
## 1.HTML核心
### 1.1 HTML 语义化标签与使用规范
**语义化标签**：用正确的标签做正确的事，让页面的内容结构化，便于浏览器、搜索引擎解析和源码阅读。

**HTML5 新增的语义化标签**：
- 布局标签
    |标签|标签名|描述|
    |:--:|:--:|:--:|
    |`<header>`| 头部标签 |定义文档或节的页眉。|
    | `<nav>`| 导航标签 |定义导航链接的容器。|
    | `<article>`| 内容标签 |定义独立的文章或内容。|
    | `<section>`| 节标签 |定义文档中的某个区域（section、chapter、part、或任意其他内容）。|
    | `<aside>`| 侧边栏标签 |定义页面的侧边栏内容。|
    | `<footer>`| 底部标签 |定义文档或节的页脚。|

- 视频标签 `<video>`
    |属性|描述|
    |:--:|:--:|
    |`src`|指定视频文件的 URL。|
    |`controls`|显示视频播放控件（播放、暂停、音量等）。|
    |`width`|指定视频的宽度（单位：像素）。|
    |`height`|指定视频的高度（单位：像素）。|
    |`autoplay`|视频加载后自动播放。|
    |`loop`|视频播放结束后循环播放。|
    |`muted`|视频默认静音播放。|

- 音频标签 `<audio>`
    |属性|描述|
    |:--:|:--:|
    |`src`|指定音频文件的 URL。|
    |`controls`|布尔属性，显示音频播放控件（播放、暂停、音量等）。|
    |`autoplay`|布尔属性，音频加载后自动播放。|
    |`loop`|布尔属性，音频播放结束后循环播放。|
    |`muted`|布尔属性，音频默认静音播放。|
    |`preload`|定义音频的预加载方式，可选值 auto：自动预加载音频;metadata：仅加载音频元数据（如时长、采样率）;none：不预加载音频|

- 图片标签 `<img>`
    |属性|描述|
    |:--:|:--:|
    |`src`|指定图片文件的 URL。|
    |`alt`|指定图片的替代文本，当图片无法显示时显示。|
    |`width`|指定图片的宽度（单位：像素）。|
    |`height`|指定图片的高度（单位：像素）。|
    |`loading`|指定图片的加载方式（`lazy` 或 `eager`）。|
    |`decoding`|指定图片的解码方式（`sync` 或 `async`）。|
    |`srcset`|指定多个图片文件的 URL，用于响应式图片。|
    |`sizes`|指定不同图片文件的显示宽度，用于响应式图片。|
    |`crossorigin`|指定图片的跨域请求方式（`anonymous`匿名跨域 或 `use-credentials`）带凭证跨域。|
    |`usemap`|指定图片的（`<map>` 元素），用于点击区域跳转。|
    |`ismap`|布尔属性，指定图片的（`<map>` 元素），若存在则图片为服务器端映射，点击图片会向服务器发送点击坐标。|
    |`class/id`|指定图片的样式类名或唯一标识符。|


### 1.2. HTML5 新特性（多媒体 / 画布 / 存储 / 语义标签）
- **语义标签**：`<header>`、`<nav>`、`<article>`、`<section>`、`<aside>`、`<footer>`
- **多媒体**：`<audio>`、`<video>`标签
- **画布**：`<canvas>`标签及其API
- **存储**：`localStorage`本地离线存储API、`sessionStorage`会话离线存储API
- 其他：`Drag and drop`拖拽API、`Geolocation`地理API、表单控件（`calendar、date、time、email、url、search）

### 1.3. canvas 与 svg 的核心区别与使用场景
||**canvas**|**svg**|
|:--:|--|--|
|**核心区别**|基于h5新增的API使用JS进行的像素级的绘制，适合动态变化的图形（如游戏、数据可视化）|基于html标签空进行矢量的绘制，适合静态图像（如图标、图表）|
|**使用场景**|游戏开发、数据可视化、实时渲染|静态图像、图标、图表|
|**颜色支持**|多|少|
|**变更性**|无法对已绘制的图像进行修改、操作|可以获取标签进行操作|


### 1.4. HTML 全局属性（data-*/draggable 等）
- class :为元素设置类标识
- data-* : 为元素增加自定义属性
- draggable : 设置元素是否可拖拽
- id : 元素 id ，文档内唯一
- lang : 元素内容的的语言
- style : 行内 css 样式
- title : 元素相关的建议信息

### 1.5. alt 与 title 属性的区别与使用规范
||**alt**|**title**|
|:--:|--|--|
|**核心区别**|alt属性在图片无法显示时用于替代图片的文本|title属性用于提示，鼠标悬停时显示|
|**使用场景**|只能用与`<img`|除`base` `basefont` `head` `html` `meta` `param` `script` `title`之外的所有标签|

## 2. CSS 核心
### 2.1. 盒模型（标准盒 / IE 盒）与 box-sizing 属性
||**标准盒（W3C盒模型）**|**怪异盒（IE盒模型）**|
|:--:|--|--|
|**核心区别**|标准盒模型：宽度和高度包含 padding 和 border|IE 盒模型：宽度和高度不包含 padding 和 border|
|**`box-sizing`设置**|`content-box`：默认值，border 和 padding 不算到 width 范围内，可以理解为是 W3c 的标准模型(default)。总宽=width+padding+border+margin|`border-box`：border 和 padding 划归到 width 范围内，可以理解为是 IE 的怪异盒模型，总宽=width+margin|

### 2.2. 选择器类型、优先级计算规则
以下按优先级排序
1.  `.class`内使用 `!important` 为最高
2. 内联样式 `style=""`
3. id选择器 `#header`
4. 伪类选择器 `:hover`
5. 属性选择器 `[type="text"]` `[href]`
6. 类选择器 `.class`
7. 伪元素选择器 `::before` `::after` `::first-line`
8. 元素选择器 `div` `p`

!important > 内联样式 > ID > 类/伪类/属性 > 元素/伪元素 > 继承 > 默认样式

**ID 的权重永远高于类**：无论有多少个类，都比不过一个 ID。

### 2.3. 浮动布局原理、优缺点与清除浮动方案
- float 属性最初设计用于实现文字环绕图片的效果，后来被广泛应用于网页布局。其核心原理是：

    - 设置 float: left/right 的元素会脱离标准文档流，但未完全脱离文本流（文字会环绕浮动元素）。
    - 浮动元素会向左或向右移动，直到其边缘碰到包含块（父元素）的边缘或另一个浮动元素的边缘。
    - 浮动元素不再占据父元素的高度，导致父元素高度塌陷（即父元素无法被浮动子元素撑开）。
    - 如果一行无法容纳多个浮动元素，后续浮动元素会自动换行。

- 优点：
    - 文字环绕效果：实现图文混排（如文章配图）非常自然
    - 块级元素并排：使块级元素能够水平排列，打破了块级元素独占一行的限制
    - 简单灵活：通过控制浮动方向和宽度，可快速搭建多列布局

- 缺点：
    - 高度塌陷:脱离文档流，父级元素高度将塌陷
    - 布局错乱：若浮动元素宽度总和超过父容器，会导致换行错位

- 清除浮动：
    - 元素内新增额外的标签`<div style="clear:both"></div>`
    - (触发BFC)父级添加`overflow`属性，`overflow:hidden` `overflow:auto`均可
    - 父级元素设置高度
    - 使用伪类选择器 `:after`
        ```css
            .parent:after{
                content:'';
                display:block;
                height:0;
                visibility:hidden;
                clear:both
            }
        ```

### 2.4. BFC（块级格式化上下文）触发条件、渲染规则与应用场景
- BFC
    - Web 页面 CSS 视觉渲染的一部分，它决定了块级盒子的布局及浮动元素如何交互
    - 一个**独立的渲染区域**
    - **内部元素的布局不会影响外部的元素**

- 触发条件
    - 根元素 `<html>`
    - 浮动元素 **`float`** 不为 none（left / right）。
    - 绝对定位元素 `position 为 absolute 或 fixed`
    - display 值为以下之一：
        - inline-block
        - table-cell（表格单元格）
        - table-caption（表格标题）
        - flex / inline-flex（弹性布局）
        - grid / inline-grid（网格布局）
        - **flow-root**（专门创建无副作用的 BFC）
    - **overflow 不为 visible：hidden / auto / scroll**。
    - contain 值为 layout、content 或 paint。
    - 多列容器：column-count 或 column-width 不为 auto（需注意浏览器兼容性）。

- 渲染规则
    - 内部块级盒子垂直排列
    - 垂直方向的边缘重叠
    - 不与浮动元素的box重叠
    - BFC 是一个独立的容器，外面的元素不会影响里面的元素
    - 计算BFC高度时，浮动元素与会参与计算（即 BFC 可以清除内部浮动，解决高度塌陷）。

- 应用场景
 - **清除内部浮动**（解决父元素高度塌陷）
    ```css
    .parent {
        overflow: hidden;  /* 触发 BFC */
    }
    /* 或使用更语义化的方式 */
    .parent {
        display: flow-root;
    }
    ```
 - **避免垂直外边距折叠**
 - **实现自适应**两列布局（防止文字环绕浮动元素）
 - 防止元素被浮动元素覆盖
